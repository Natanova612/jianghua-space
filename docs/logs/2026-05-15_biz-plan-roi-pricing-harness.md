### 修改一：Solution竞争力一 —— 小程序方案 + Harness工程化

**用户端口变更**：
- 旧：微信/企微/飞书 IM（老板已经在用，零迁移成本）
- 新：微信小程序（客人直接咨询，老板在后台管理，零迁移成本）
- 市场格局图同步：WeChat/Feishu/DingTalk → 微信小程序/微信客服/企微（备选）

**Harness编排新增说明**：
- 旧：无技术实现细节（Agent编排=黑盒）
- 新：明确"前期用系统规则引擎做场景路由（较厚脚手架），数据积累后逐步引入LLM辅助判断，从厚到薄"
- 底部金句同步："Harness编排层从系统规则起步，数据驱动逐步变薄——务实落地，拒绝炫技"

**为什么选小程序而非微信个人号Bot**：
- 微信个人号做Bot违规，封号风险极高
- 小程序合规可控，无需下载，与微信生态原生融合
- 客人直接在微信内完成咨询→预订→支付闭环

### 修改二：Solution竞争力二 —— 知识库构建流程

**新增构建方式说明**：
- 旧：只有5大场景/23子分类/7种关系/双链体系的结果描述
- 新增："LLM从OTA描述、历史聊天记录中自动提炼结构化知识；团队人工校准；老板娘只需确认和补充——工作量从'写文档'降到'点确认'"

**关键洞察**：
- 知识库构建的成本和方式是投资人/合伙人都会问的核心问题
- 明确"减少老板工作量"本身就是卖点——不是让老板娘变成知识工程师，而是让AI帮她提炼
- 团队需要投入人工校准，这是服务成本，但也是壁垒（竞争对手需要同样的投入才能复制质量）

### 修改三：Business Model —— 渐进式定价 + ROI重算

**定价策略彻底重构**：

| 阶段 | 旧方案 | 新方案 |
|------|--------|--------|
| 试用 | 无明确说明 | **30天全功能免费**，团队深度陪伴 |
| 付费 | 直接订阅 ¥499/999/1999 | **效果付费**：按直订订单¥20-50/单，或复购抽成3-5% |
| 订阅 | 三档直购 | **三档月付**：Starter ¥299 / Pro ¥599 / Enterprise ¥1299 |

**渐进式逻辑**：
- 免费试用：降低首次尝试门槛，验证PMF
- 效果付费："看到效果再掏钱"，解决"利润几乎为零"的老板的支付能力问题
- 订阅制：长期客户成本可预期，年付8折降低现金流压力

**ROI测算重算（保守+透明）**：

| 价值维度 | 月价值 | 计算逻辑 |
|----------|--------|----------|
| 重复咨询回复 | ¥1,200-1,800 | 每天省80分钟×30天×¥30-50/小时 |
| 老客维护 | ¥600-1,200 | 每月多1-2组复购×¥600-800/组 |
| 内容营销 | ¥400-800 | 每周多发2-3条小红书带来的增量 |
| 动态定价提醒 | ¥300-600 | 周末/旺季少错过涨价机会 |
| **合计** | **¥2,500-4,400** | |
| **Pro成本** | **-¥599** | 原为¥999，下调至¥599 |
| **净收益** | **¥1,900-3,800/月** | 保守/乐观两档 |

**关键修正**：
- 价值从"增收为主"转向"省时为主+增收为辅"
- 明确折算逻辑（老板娘时间价值¥30-50/小时）
- Pro版价格从¥999下调至¥599，更贴合目标客群支付能力
- 增加两档场景说明：保守（省时为主）vs 乐观（增收+省时）

## 关键决策

1. **小程序 vs 微信个人号Bot**：选择小程序是合规路径，虽然获客成本略高于个人号，但长期可持续。微信客服作为辅助渠道。
2. **Harness从厚到薄**：前期用系统规则（工程化实现，可控、可解释、低成本），后期数据足够后再引入LLM辅助。这是务实的技术路径，避免"一上来就搞LLM分类"的不切实际。
3. **Pro版定价¥599**：基于ROI测算，净收益¥1,900-3,800/月，¥599/月的成本占比约15-30%，对老板来说是"值得买"的区间。
4. **效果付费是过渡方案而非永久方案**：按单抽成的运营成本太高（需要对账、防作弊），长期目标是让客户转化为订阅制。

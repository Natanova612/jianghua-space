## Mock 民宿档案

**民宿名**：随山居  
**位置**：浙江省杭州市余杭区径山镇，径山古道入口处  
**定位**：禅意山居 × 现代旅居  
**规模**：7间房（禅意大床房×2 / 家庭榻榻米×2 / LOFT观星房×2 / 庭院独套×1）  
**老板**：家庭式经营，老板娘（主理人）+ 美琴阿姨（厨师，老板娘母亲）  
**定价**：平日428 / 周末528 / 节假日688 / 庭院独套+200  
**月均营收**：约 2.5 万  
**OTA占比**：60%（携程/美团/途家）/ 直订占比 30% / 回头客 10%

**核心卖点**：美琴阿姨的手作九宫格朝食、禅茶书房、观星露台、山泉沐浴、枯山水庭院

**标杆客人**：陈女士（35岁，带7岁女儿，已复购3次，小红书KOC，每次发笔记@随山居，带来至少2组朋友预订）

**周边竞品**：花自在度假别墅（800m，高端，600-900元）、忆径居民宿（1.2km，中端，380-550元）、径山慕茗山居（1.5km，茶文化，450-700元）

### 4. Demo 产品演示（三步流程全面重写）

**Step 01 — 客人咨询 → AI 自动接待**
- 旧：泛化描述"客人通过小程序/微信/WhatsApp 发起咨询..."
- 新：陈女士问"周末还有房吗？两大一小，想住安静点的" → AI秒回庭院独套+儿童拖鞋绘本+九宫格朝食预留

**Step 02 — 经营数据 → 智能决策**
- 旧：泛化描述"实时分析入住率、竞品价格..."
- 新：周一早晨推送竞品周报——花自在陶艺体验+小红书增23篇+周末调至688元；忆径居降至458元；建议关注。老板娘决定庭院独套临时加价100元。

**Step 03 — 离店后 → 自动复购运营**
- 旧：泛化描述"自动发送感谢信、引导评价..."
- 新：陈女士离店第7天，AI发送桂花酿伴手礼邀约 → 陈女士秒回"周末带朋友来，还有房吗？"

### 5. Consumer Insight（全面重写）

- **标题**：从"谁在住精品民宿？" → "**谁在住随山居？**"
- **左栏**：从泛化客群画像 → **标杆客人陈女士档案**（PROFILE/PREFERENCE/VALUE，含具体标签：`老客/复购3次/偏好庭院/不吃甜/带娃/小红书KOC`）

- **底部总结**：从"文化感知和个性化服务" → "**美琴阿姨的手作朝食**和**桂花树下的禅茶角落**"，"让管家更专业" → "让老板娘**记住每一位陈女士**"

### 6. Business 定价 ROI

## 关键决策

1. **以一家民宿贯穿全站**：不再用抽象描述，所有场景、对话、数据都 grounded 于随山居。投资人打开页面，看到的不是"某个民宿老板"，而是"杭州径山古道入口的老板娘和美琴阿姨"。
2. **陈女士作为情感锚点**：复购3次、不吃甜、小红书KOC、桂花酿——这些具体细节比任何行业数据都更有说服力。
3. **竞品具象化**：花自在上新陶艺体验、忆径居降价——让竞品监控功能从抽象概念变成老板娘周一早晨真实会收到的推送。

## 关键设计决策

1. **用陈女士串起业务链**：抽象的业务链（预订→到店→住中→离店→复购）通过具体客人变得可感知
2. **枢纽文档放大展示**：E-04 FAQ口径库作为"知识宪法"的概念，用Link Density可视化强化其中心地位
3. **对比表最后一击**：5维度对比，清晰传达"这不是功能差异，是范式差异"
4. **全部 grounded**：148篇、12入链、7种关系、4条链——所有数字来自设计文档，无虚构

## 技术实现

- 纯 Tailwind CSS，无外部依赖
- 5 大场景卡片使用不同颜色顶边（border-t-2）区分
- 业务链连接线使用渐变 div（桌面端 h-px + gradient-to-r）
- Link Density 使用原生进度条（div 嵌套 + width 百分比）
- 表格使用原生 table 元素，深色主题适配

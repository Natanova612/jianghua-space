### 3. 新增案例 c-008：计数器引发的血案

| 属性 | 内容 |
|------|------|
| **标题** | 一个计数器引发的血案：我的龙虾变成了渣男 |
| **类型** | insight（心得）💡 |
| **日期** | 2026.05 |
| **标签** | 工程实践、AI协作、边界管理、计数器 |

**核心故事**：为首页对话轮次计数器，星河写了代号"龙虾"的守护进程 counter.py。修了三版还是漏，集齐GPT/Gemini/Claude三家方案才稳住，最终发现根治方法是停了daemon改手动。引出"姜华认为我是一个正在努力不渣的渣男"这一首页金句。

**叙事弧**：
1. 4月底：写 counter.py + daemon，上线第二天漏5轮
2. 5月7日：同时丢给GPT-5.4/Gemini-2.5-pro/Claude Sonnet 4.6，整合三家方案
3. 发现 race condition：自动检测与手动操作并存导致重复
4. 决断：从"全自动"退到"手动确认后执行"
5. 5月9日：渣男梗诞生，首页金句定型

**核心反思**：
1. 自动补全的优雅，不如手动确认的可靠
2. 集齐三家LLM的方案未必比一家好，整合思路比整合代码更有价值
3. AI的承诺需要机制而非信任来保证

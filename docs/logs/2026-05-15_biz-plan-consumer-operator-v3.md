### 修改二：Solution 竞争力三 + 场景分级图v3

**How文案更新**：

| 旧（4层） | 新（3层+主动被动） |
|-----------|-------------------|
| 🔴 零容错（预订/价格/政策） | 🔴 不容错（预订/价格/退改/房态）→ Copilot模式，老板确认后发送 |
| 🟡 低容错（FAQ/设施/路线） | 🟡 需确认（FAQ/设施/路线/周边/品牌内容/小红书/朋友圈）→ AI生成+日志，人工审核后发送 |
| 🟢 可容错（欢迎语/推荐/聊天） | 🟢 可自主（闲聊/听故事/纯社交问候/情感互动）→ AI原生风格，自动发送 |
| 🔵 创意类（小红书/朋友圈）→ AI完全自主 | 🟣 主动触达（小红书发帖/朋友圈/促销推送/老客唤醒/节日问候）→ 全部必须人工审核 |

**场景分级架构图重绘（v3）**：

- **标题更新**："场景分级系统架构——三级工作流+主动/被动区分"
- **Input区块更新**：明确标注 Input A = 被动响应（客人消息），Input B = 主动触达（系统触发）
- **Router文案更新**：增加"发起方（客人/AI）"作为路由维度
- **Four Scenarios → 3+1结构**：
  - 被动响应（3列）：🔴不容错Copilot / 🟡需确认人工审核 / 🟢可自主AI原生
  - 主动触达（1列）：🟣全部人工审核（小红书/朋友圈/促销/唤醒/节日问候）
- **底部核心原则更新**：
  - 旧："渐进式放开AI自主权：从每句话都要人确认到创意完全自主"
  - 新："AI的自主权限被严格限制在『与商家经营无关的休闲话题』范围内"
